# mls-sim.exe 使用说明

## 简介

`mls-sim.exe` 是 MLS 云脚本本地模拟器的可执行文件。合并上游 `0.4.3` 后，程序使用原生 GUI 管理房间、日志、出站事件、状态和设置，同时保留 Bridge HTTP 接口供客户端轮询和发送事件。

## 获取方式

在 `mls-sim-rs` 目录下编译：

```powershell
cd mls-sim-rs
cargo build --release
```

编译完成后，可执行文件位于：

```text
mls-sim-rs\target\release\mls-sim.exe
```

## 启动

```powershell
mls-sim.exe --script-dir "D:/你的云脚本目录/script"
```

启动后会自动创建房间、加载 `main.lua`，并打开原生 GUI 窗口。

## 命令行参数

| 参数 | 缩写 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `--host` | 无 | `127.0.0.1` | 监听地址 |
| `--port` | `-p` | `5000` | 监听端口 |
| `--script-dir` | `-s` | 无 | 云脚本目录（目录下需有 `main.lua`） |
| `--config` | 无 | `config.json` | 配置文件路径 |
| `--console-notwrte` | 无 | `false` | Windows 下隐藏控制台窗口，仅保留 GUI 窗口 |

## 用法示例

```powershell
# 最简启动
mls-sim.exe -s "D:/Code/MlScriptFolder/script"

# 换端口
mls-sim.exe -s "D:/script" -p 5001

# 用配置文件（可预设玩家、道具、存档）
mls-sim.exe --config "D:/my_config.json"

# Windows 下隐藏控制台窗口
mls-sim.exe -s "D:/script" --console-notwrte

# 空启动，在 GUI 里手动建房
mls-sim.exe
```

## 原生 GUI

启动后会打开 `MLS 云脚本环境模拟` 窗口：

- 房间：创建、启动、停止、销毁房间，管理玩家并发送测试事件。
- 控制台：查看脚本日志，支持级别、房间和文本过滤。
- 出站事件：查看云脚本发给客户端的事件。
- 状态：查看房间状态 JSON。
- 设置：查看和保存监听地址、端口、存档目录等配置。

Bridge HTTP 服务仍会监听 `--host` 和 `--port` 指定的地址，客户端可继续通过 `/api/bridge/login`、`/api/bridge/event`、`/api/bridge/poll/{room_id}/{player_index}` 对接。

## 存档

- 房间停止、GUI 关闭或进程正常退出时会保存到 `archives/` 目录
- 下次创建同名脚本房间时自动读取上次存档
- 强制结束进程可能来不及保存，优先使用 GUI 停止房间或正常关闭窗口

## 常见问题

| 问题 | 解决 |
| --- | --- |
| 端口占用 | `--port 5001` 换端口 |
| GUI 没显示 | 确认程序没有被安全软件拦截；Windows 下不要误用隐藏窗口工具启动 |
| 脚本没加载 | 检查目录下有没有 `main.lua` |
| 存档没保存 | 优先点“停止”或正常关闭 GUI，避免强制结束进程 |
| 日志 LOG FUSE | 日志太频繁被限流，减少输出 |
