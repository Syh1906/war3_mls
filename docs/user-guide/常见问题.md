# 常见问题

> Parent: [README.md](README.md)

## 编译失败

确认 Rust 工具链已安装：

```powershell
rustup --version
cargo --version
```

推荐 stable 最新版。如果报 C++ 编译器相关错误，确认安装了 Visual Studio Build Tools（`rustup` 通常会提示安装）。

## 启动后浏览器没有打开

确认配置文件中 `auto_open_browser` 为 `true`，或手动打开：

```
http://127.0.0.1:5000
```

## 端口被占用

换一个端口：

```powershell
mls-sim.exe --port 5001
```

## 云脚本没有加载

检查脚本目录下是否有 `main.lua` 文件。模拟器固定加载入口文件 `main.lua`。

查看 Web 面板的日志区域，如果有红色 ERR 日志，说明脚本加载或执行出错。

## 存档没有保存

存档在房间停止时自动保存。需要调用 `POST /api/rooms/{room_id}/stop` 或在 Web 面板点"停止"。

直接关闭模拟器进程不会触发保存。

## 存档没有自动加载

确认上次停止房间时存档已保存到 `archives/` 目录。文件名为 `<脚本目录名>.json`。

创建房间时会自动查找同名存档文件并加载。如果配置文件中手动指定了存档数据，手动值优先。

## WebSocket 显示"未连接"

Web 面板使用原生 WebSocket 连接。确认模拟器正在运行且端口正确。WebSocket 断线后会自动每 2 秒重连。

## 中文路径问题

Rust 原生支持 Unicode 路径，中文路径可以直接使用，不需要特殊处理。

## 日志被熔断了

模拟器与平台一致，限制 100 秒内最多 1000 条日志。减少日志输出频率，或只在关键位置打印。

## Bridge 轮询返回空数组

1. 确认房间状态是"运行中"
2. 确认 `player_index` 正确
3. 轮询会清空队列，连续轮询空队列就是空数组
4. 先发一个事件让脚本产生响应

## 多客户端怎么测试

多个客户端使用不同的 `player_index` 连接同一个房间：

```python
# 客户端 A
requests.post(url, json={"room_id": "room-001", "player_index": 0, ...})

# 客户端 B
requests.post(url, json={"room_id": "room-001", "player_index": 1, ...})
```

在配置文件的 `players` 数组中预先定义好对应数量的玩家。
