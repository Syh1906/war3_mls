# 快速开始


## 环境要求

- Rust 工具链（[rustup](https://rustup.rs/) 安装，选 stable 即可）
- Windows 10 或更高版本

## 编译

```powershell
cd mls-sim-rs
cargo build --release
```

产物位于 `target/release/mls-sim.exe`，约 5MB，不依赖任何外部 DLL。

## 启动

### 最简启动（指定脚本目录）

```powershell
mls-sim.exe --script-dir "D:/你的脚本目录/script"
```

启动后：
- 自动创建一个房间
- 加载脚本目录下的 `main.lua`
- 打开原生 GUI 窗口

### 使用配置文件

```powershell
copy config.example.json config.json
# 编辑 config.json，修改 script_dir 等路径
mls-sim.exe
```

### 空启动（手动建房）

```powershell
mls-sim.exe --port 5000
```

在原生 GUI 的“房间”页点击“新建”手动创建房间。

## 验证是否启动成功

```powershell
curl http://127.0.0.1:5000/api/health
```

正常返回：

```json
{
  "ok": true,
  "name": "mls-sim",
  "version": "0.4.3",
  "room_count": 1
}
```

## 发送测试事件

通过 curl 向云脚本发送事件：

```powershell
curl -X POST http://127.0.0.1:5000/api/bridge/event -H "Content-Type: application/json" -d "{\"room_id\":\"room-001\",\"player_index\":0,\"ename\":\"testapi\",\"evalue\":\"\"}"
```

也可以在原生 GUI 的“房间”页发送事件区域直接操作。

## 模拟客户端轮询

```powershell
curl http://127.0.0.1:5000/api/bridge/poll/room-001/0
```

返回云脚本发给该玩家的所有待取事件。

## 命令行参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--host` | `127.0.0.1` | 监听地址 |
| `--port` / `-p` | `5000` | 监听端口 |
| `--script-dir` / `-s` | 无 | 云脚本目录，启动后自动建房 |
| `--config` | `config.json` | 配置文件路径 |
| `--console-notwrte` | `false` | Windows 下隐藏控制台窗口，仅保留 GUI 窗口 |
