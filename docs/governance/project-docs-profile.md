## 项目定位

- 项目一句话定位：这是一个用 Rust 实现的 MLS 云脚本本地模拟器，用于在本机复现云脚本运行环境、客户端连接和调试链路。
- 正式 docs 主要服务哪些读者：云脚本作者、模拟器维护者、客户端对接者，以及需要在仓库内持续维护正式文档的协作者和 Agent。
- 当前 docs 治理的目标：把项目正式文档统一纳入可托管的规范、模板、计划和 docs impact review 流程，同时保留个人 Fork 维护所需的治理入口。

## README 规则

- `docs/` 体系 README 是否默认采用导航型：是，固定顺序为标题、顶部定位说明、`给 LLM 的快速索引`、`给人类的导航`。
- 代码目录 README 是否采用目录索引型：按需启用；当前治理层未要求业务代码目录统一落盘目录索引 README。
- 哪些目录已有稳定 README 样式：`docs/` 根目录、`docs/standards/`、`docs/templates/`、`docs/governance/`、`docs/plans/`、`docs/plans/archive/` 及已启用业务文档目录均采用导航型 README；`> Parent:` 没有登记为稳定样式。

## 风格约束

- 稳定表头：`主题 / 直接入口 / 读取说明`、`文件 / 作用`、`目录 / 作用 / 是否启用`、`日期 / 变更`、`编号 / 计划 / 完成日期 / 回填说明`。
- 禁词：简单、显然、强大、优雅、完美、深入探讨、赋能、全面。
- 术语统一要求：MLS、云脚本、模拟器、房间、玩家、Bridge、docs impact review。
- 图示最低要求：新增长期说明或大幅重写时，正文前半段至少有 1 个图示或结构矩阵。

## docs impact review

- 哪些改动必须触发 docs impact review：代码、配置、脚本、接口、UI、行为、目录结构、测试口径和计划状态变化。
- 命中后最少检查哪些 docs 资产：当前正文、所在目录 `README.md`、`docs/README.md`、`docs/standards/*.md`、`docs/templates/*.md`、`docs/plans/README.md`、`docs/plans/archive/README.md`、API 文档、用户指南、`docs/governance/*.md` 和 `docs/llm/*.md`。
