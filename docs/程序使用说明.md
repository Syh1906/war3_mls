# mls-sim.exe 使用说明

## 简介

`mls-sim.exe` 是 MLS 云脚本本地模拟器的可执行文件，单文件即可运行，不依赖任何外部 DLL 或运行时。

启动后会在本地开启一个 HTTP 服务，自带 Web 控制面板，模拟平台云脚本的完整运行环境。

---

## 获取方式

在 `mls-sim-rs` 目录下编译：

```powershell
cd mls-sim-rs
cargo build --release
```

编译完成后，可执行文件位于：

```
mls-sim-rs\target\release\mls-sim.exe
```

约 5MB，可以复制到任意目录使用。

---

## 基本用法

### 最简启动

```powershell
mls-sim.exe --script-dir "D:/你的云脚本目录/script"
```

效果：
- 在 `127.0.0.1:5000` 启动 HTTP 服务
- 自动创建一个房间，加载指定目录下的 `main.lua`
- 自动打开浏览器进入 Web 控制面板
- 默认创建 1 个玩家（槽位 0）

### 指定端口

```powershell
mls-sim.exe --script-dir "D:/script" --port 5001
```

### 使用配置文件

```powershell
mls-sim.exe --config "D:/my_config.json"
```

配置文件可以预设玩家数量、道具、存档等，适合固定的测试场景。

### 空启动（不自动建房）

```powershell
mls-sim.exe
```

启动后打开 `http://127.0.0.1:5000`，在 Web 面板手动创建房间。

## 命令行参数

```
mls-sim.exe [参数]
```

| 参数 | 缩写 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `--host` | 无 | `127.0.0.1` | HTTP 服务监听地址 |
| `--port` | `-p` | `5000` | HTTP 服务监听端口 |
| `--script-dir` | `-s` | 无 | 云脚本目录路径（目录下需有 `main.lua`） |
| `--config` | 无 | `config.json` | 配置文件路径 |

示例：

```powershell
# 监听所有网卡，端口 8080
mls-sim.exe --host 0.0.0.0 --port 8080 --script-dir "D:/script"

# 使用自定义配置
mls-sim.exe --config "D:/test/config.json"

# 使用缩写
mls-sim.exe -s "D:/script" -p 5001
```

## 配置文件详解

配置文件为 JSON 格式，所有字段都可选，未填写的使用默认值。

### 完整示例

```json
{
  "host": "127.0.0.1",
  "port": 5000,
  "auto_open_browser": true,
  "archive_dir": "./archives",
  "auto_room": {
    "script_dir": "D:/Code/MlScriptFolder/script",
    "mode_id": 0,
    "players": [
      {
        "index": 0,
        "name": "玩家1",
        "map_level": 5,
        "map_exp": 1200,
        "played_count": 10,
        "items": {
          "VIP001": 1,
          "GOLD_CARD": 3
        },
        "script_archive": "{\"level\":5,\"gold\":9999}",
        "common_archive": {},
        "read_archive": {
          "boss_kill": "15"
        },
        "cfg_archive": {}
      },
      {
        "index": 1,
        "name": "玩家2"
      },
      {
        "index": 2,
        "name": "玩家3"
      }
    ]
  }
}
```

### 字段说明

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `host` | string | `"127.0.0.1"` | 监听地址 |
| `port` | number | `5000` | 监听端口 |
| `auto_open_browser` | bool | `true` | 启动后自动打开浏览器 |
| `archive_dir` | string | `"./archives"` | 存档文件保存目录 |
| `auto_room` | object | 无 | 启动后自动创建的房间（不填则不自动建房） |

#### auto_room 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `script_dir` | string | 云脚本目录路径，目录下必须有 `main.lua` |
| `mode_id` | number | 游戏模式 ID，对应 `MsGetRoomModeId()` |
| `players` | array | 玩家列表 |

#### players 数组中每个玩家的字段

| 字段 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `index` | number | `0` | 玩家槽位编号 |
| `name` | string | `"Player_0"` | 玩家昵称 |
| `map_level` | number | `1` | 地图等级 |
| `map_exp` | number | `0` | 地图经验 |
| `played_count` | number | `1` | 游玩次数 |
| `items` | object | `{}` | 道具，格式为 `{"道具ID": 数量}` |
| `script_archive` | string | 无 | 脚本存档数据（JSON 字符串） |
| `common_archive` | object | `{}` | 普通存档 `{"key": "value"}` |
| `read_archive` | object | `{}` | 只读存档 `{"key": "value"}` |
| `cfg_archive` | object | `{}` | 全局只读存档 `{"key": "value"}` |

### 最简配置

只需要指定脚本目录，其余全部使用默认值：

```json
{
  "auto_room": {
    "script_dir": "D:/Code/MlScriptFolder/script"
  }
}
```

## 启动后的操作

### Web 控制面板

浏览器访问 `http://127.0.0.1:5000`（或你指定的端口），可以：

| 功能 | 说明 |
| --- | --- |
| 查看房间列表 | 左侧边栏显示所有房间及状态 |
| 创建新房间 | 点击"+ 新建"，填写脚本目录和玩家 |
| 启动/停止/销毁房间 | 右上角操作按钮 |
| 查看玩家状态 | 显示在线/离线、等级、道具数量 |
| 模拟玩家断线/重连/退出 | 每个玩家卡片上的按钮 |
| 发送事件 | 选择事件名、数据和目标玩家，点击发送 |
| 查看实时日志 | 底部日志面板，支持级别过滤和关键字搜索 |
| 查看出站事件 | 云脚本通过 MsSendMlEvent 发出的事件 |
| 查看房间状态 JSON | 状态查看面板，点击刷新 |

### 命令行验证

```powershell
# 检查服务状态
curl http://127.0.0.1:5000/api/health

# 发送事件给云脚本
curl -X POST http://127.0.0.1:5000/api/rooms/room-001/events ^
  -H "Content-Type: application/json" ^
  -d "{\"ename\":\"testapi\",\"evalue\":\"\",\"player_index\":0}"

# 轮询云脚本返回的事件
curl http://127.0.0.1:5000/api/bridge/poll/room-001/0

# 停止房间（触发存档保存）
curl -X POST http://127.0.0.1:5000/api/rooms/room-001/stop
```

## 存档

### 自动保存

房间停止时（点击"停止"或调用 stop 接口），自动将所有玩家的存档数据保存到：

```
<archive_dir>/<脚本目录名>.json
```

默认位置举例：`./archives/script.json`

### 自动读取

下次启动并创建同名脚本目录的房间时，自动从存档文件读取上次保存的数据，包括：
- 脚本存档（script_archive）
- 普通存档（common_archive）
- 只读存档（read_archive）
- 全局只读存档（cfg_archive）
- 道具（items）
- 等级、经验、游玩次数

如果配置文件中手动指定了某个字段，手动值优先，不会被存档覆盖。

### 存档文件格式

```json
{
  "0": {
    "name": "玩家1",
    "map_level": 5,
    "items": {"VIP001": 1},
    "script_archive": "{\"level\":5}",
    "common_archive": {},
    "read_archive": {"boss_kill": "15"},
    "cfg_archive": {},
    "saved_at": "2026-04-24 15:30:00"
  },
  "1": {
    "name": "玩家2",
    ...
  }
}
```

## 多客户端测试

模拟多个客户端连接同一个房间，每个客户端使用不同的 `player_index`：

配置文件中预设多个玩家：

```json
{
  "auto_room": {
    "script_dir": "D:/script",
    "players": [
      {"index": 0, "name": "主机玩家"},
      {"index": 1, "name": "客户端A"},
      {"index": 2, "name": "客户端B"},
      {"index": 3, "name": "客户端C"}
    ]
  }
}
```

每个测试客户端用对应的 `player_index` 发送和轮询事件。

## 常见问题速查

| 问题 | 解决 |
| --- | --- |
| 启动报端口占用 | 用 `--port 5001` 换端口 |
| 浏览器没自动打开 | 手动访问 `http://127.0.0.1:5000` |
| 脚本没加载 | 检查脚本目录下是否有 `main.lua` |
| 存档没保存 | 需要点"停止"或调用 stop 接口，直接关进程不会保存 |
| 日志显示 LOG FUSE | 日志太频繁被限流了，减少日志输出 |
| 中文路径报错 | 不应该报错，Rust 原生支持中文路径 |
