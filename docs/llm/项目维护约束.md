# 项目维护约束

> Parent: [README.md](README.md)

## 适用场景

这些约束适用于 Agent 修改代码、配置和正式 docs 时。

## 稳定约束

| 约束 | 原因 |
| --- | --- |
| `origin` 是个人 Fork，`upstream` 是原作者仓库 | 日常开发推到自己的 Fork，贡献上游时另开干净分支 |
| `.local/` 是本机资料目录 | 本机备忘不进入 Git |
| `.vscode/`、`.env`、`mls-sim-rs/config.json` 不提交 | 这些内容通常带本机路径或私有配置 |
| 正式 docs 任务先遵守 `docs/standards/` | 项目已进入 docs 托管模式 |
| docs 目录结构变化必须同步父级 README | 保持导航入口可用 |

## 易错点

| 易错点 | 正确做法 |
| --- | --- |
| 从包含个人 docs 体系的 `main` 直接向上游发 Pull Request | 从 `upstream/main` 新建贡献分支，再挑选提交 |
| 迁移文档后只改文件路径，不改内部链接 | 同步修正 README 和正文中的相对链接 |
| 修改接口实现后忘记文档 | 同步检查 `docs/api/`、`docs/client/` 和 `docs/user-guide/` |
