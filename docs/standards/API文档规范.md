# API文档规范

## 适用范围

- 接口契约说明文档。
- 接口变更记录。
- 与接口契约直接对应的长期说明文档。

## 真相源优先级

1. 真实 handler / 路由注册 / 请求解析代码
2. 真实调用方、测试和脚本
3. 契约产物
4. Markdown 说明文档

## 契约文档默认结构

1. `# 标题`
2. `## 元信息`
3. `## 概述`
4. `## 文档定位`
5. `## 真相源`
6. `## 文档边界`
7. `## 通用契约`
8. 图示或结构矩阵
9. 按主题展开的端点职责、边界、注意事项
10. `## 变更记录`

## 写法要求

- 解释谁该调用、调用方应怎么理解、冲突时信谁。
- 不逐字段复制契约产物。
- 顶层公共口径和跨端点共性，写在通用章节里。

## 接口变更记录格式

- 记录区按时间倒序追加。
- 每次新增记录时，在最上方追加：`### YYYY-MM-DD — 变更摘要`
- 每条记录固定三列：`路由 / 方法 / 变更内容`

## 同步规则

- 接口行为变更：同步契约说明 + 契约产物 + 接口变更记录。
- 仅实现重构、对外语义不变时，可只更新长期说明。
