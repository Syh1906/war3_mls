# LLM知识文档规范

## 适用范围

- `docs/llm/*.md` 中除 README 外的轻量知识条目与局部约束条目。

## 目标

- 给协作者和 LLM 提供高频、稳定、容易错的规则入口。
- 让单条知识可以被按需命中，而不是通读整个目录。

## 默认结构

1. `# 标题`
2. `## 触发场景`
3. `## 规则`
4. `## 相关文件`

## 写法要求

- 每条规则尽量短、小、准。
- 只写稳定知识、边界和约束，不写临时排查过程。
- 相关文件只列稳定入口，不列无意义噪音文件。

## 禁止项

- 把长篇正式文档改写成 `docs/llm` 条目。
- 记录未经验证的猜测或会快速过期的一次性结论。
