# README索引规范

## README 类型判定

当前治理体系里的 README 分两类，不能混写。

| 类型 | 适用位置 | 默认结构 |
| --- | --- | --- |
| 导航型 README | `docs/` 体系下全部 README | 顶部定位说明 + `给 LLM 的快速索引` + `给人类的导航` |
| 目录索引型 README | `docs/` 之外的代码目录、工具目录等索引页 | `目录职责` + `子目录索引` + `文件索引` |

## 导航型 README

### 固定结构

1. `# <dir> 导航`
2. 顶部一段定位说明
3. `## 给 LLM 的快速索引`
4. 主题入口表
5. `## 给人类的导航`
6. 面向人的阅读建议或入口表

### 写法要求

- 第一段要说明：本目录真正的事实源是什么、README 只负责什么。
- 入口信息优先用表格，列名优先使用 `主题 / 直接入口 / 读取说明`、`入口 / 适合什么时候看`。
- 新增、移动、删除正文后，立即同步入口表。

## 目录索引型 README

### 固定结构

1. `# <dir> 目录索引`
2. `## 目录职责`
3. `## 子目录索引`
4. `## 文件索引`

### 写法要求

- `目录职责` 至少写 1-2 条，回答目录整体负责什么。
- `子目录索引` 没有子目录时写 `- 无。`
- `文件索引` 用表格列出 `文件 / 职责 / 关键入口`。

## 禁止项

- 在 docs 导航页里混入大量正文内容。
- 在目录索引型 README 中省略 `目录职责` 或 `文件索引`。
