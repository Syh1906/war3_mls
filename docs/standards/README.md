# docs 规范入口

> Parent: [../README.md](../README.md)

本目录定义项目内正式文档的写作和同步规则。项目已进入 docs 托管模式，正式 docs 优先遵守本目录规范。

## 给 LLM 的快速索引

| 任务 | 入口 |
| --- | --- |
| 写或改任何正式 docs | [通用文档规范](通用文档规范.md) |
| 维护 `README.md` 导航 | [README 索引规范](README索引规范.md) |
| 写架构、边界、目录职责 | [长期说明文档规范](长期说明文档规范.md) |
| 写计划和归档 | [计划文档规范](计划文档规范.md) |
| 写使用指南 | [用户指南规范](用户指南规范.md) |
| 写 HTTP 或 Lua API | [API 文档规范](API文档规范.md) |
| 写 LLM 约束 | [LLM 知识文档规范](LLM知识文档规范.md) |
| 维护项目画像 | [项目画像文档规范](项目画像文档规范.md) |

## 给人类的导航

| 文件 | 作用 |
| --- | --- |
| `通用文档规范.md` | 所有正式 docs 的共通规则 |
| `README索引规范.md` | 导航型 README 的固定结构 |
| `长期说明文档规范.md` | 架构、边界和维护说明的写法 |
| `计划文档规范.md` | 计划、状态和归档的写法 |
| `用户指南规范.md` | 面向使用者的操作指南写法 |
| `API文档规范.md` | HTTP、Lua API 和接口变更记录写法 |
| `LLM知识文档规范.md` | LLM 约束和易错点写法 |
| `项目画像文档规范.md` | `project-docs-profile.md` 的维护规则 |
