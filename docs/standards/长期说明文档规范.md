# 长期说明文档规范

## 适用范围

- `docs/architecture/*.md`、`docs/server/*.md`、`docs/client/*.md`、`docs/web/*.md`、`docs/ops/*.md` 中除 README 外的长期正文。
- 用于解释架构、边界、职责、维护规则、运行语义、目录分工和长期流程的文档。

## 默认章节顺序

1. `# 标题`
2. `## 元信息`
3. `## 概述`
4. `## 文档定位`
5. `## 真相源`
6. `## 文档边界`
7. `## 关键图示`
8. `## 主题展开`
9. `## 变更记录`

## 内容约束

- 回答“边界是什么、为什么这样分、哪份文件才是真相源、修改时要注意什么”。
- 不逐字段抄契约产物，不重复一眼可见的实现细节，不写临时排查过程。
- 真相源必须落表。
- 文档边界必须明确写“不负责什么”。

## 图示要求

- 新文档或大重写时，正文前半段至少有 1 个图示或结构矩阵。
- 图示只解释稳定结构，不画临时执行痕迹。

## 禁止项

- 把教程步骤写进长期说明文档。
- 把计划、待办复选框写进长期说明文档。
- 把字段 schema 大段复制进长期说明文档。
