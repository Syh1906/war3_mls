# 项目画像文档规范

## 适用范围

- `docs/governance/project-docs-profile.md`

## 目标

- 维护项目 docs 治理的稳定画像。
- 告诉后续协作者：这个项目有哪些 docs 域、哪些类型已启用、真相源优先级是什么、docs impact review 应覆盖什么。

## 默认结构

1. `# 项目 docs 画像`
2. `## 元信息`
3. `## 项目定位`
4. `## docs 目录规划`
5. `## 真相源优先级`
6. `## README 规则`
7. `## 文档类型与同步点`
8. `## 计划治理`
9. `## 风格约束`
10. `## docs impact review`
11. `## 与 skill 内建规范的对齐策略`

## 写法要求

- 这里记录的是治理配置和稳定事实，不是普通知识条目。
- 更新时优先增量修改现有字段，不要随意改写整体结构。
- 目录规划、类型启用状态、同步点和真相源优先级变化后，必须同步这里。
- 启用 `docs/plans/` 时，必须在 `计划治理` 中记录编号分配、当前计划命名、归档命名、状态枚举、归档前置条件和索引同步规则。

## 禁止项

- 把 `project-docs-profile.md` 当成普通 `docs/llm` 约束条目改写成“触发场景 / 规则 / 相关文件”结构。
- 在这里记录一次性排查结论或临时执行痕迹。
