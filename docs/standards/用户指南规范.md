# 用户指南规范

## 适用范围

- `docs/user-guide/*.md` 的使用说明正文。
- 在线帮助系统或 guide 配置直接消费的 Markdown 正文。

## 文件头与同步点

1. 每个文件第一行必须是 `<!-- updated: YYYY-MM-DD -->`。
2. 新增章节时，同步更新目录入口和在线展示配置入口。
3. 页面、菜单、权限、操作流程变更后，必须同波次更新对应章节。

## 默认章节顺序

1. `<!-- updated: YYYY-MM-DD -->`
2. `# 标题`
3. 简短说明：这个页面或功能做什么
4. `## 使用前提` 或 `## 入口路径`
5. `## 子页面` / `## 页面结构`
6. `## 关键操作` / `## 操作步骤`
7. `## 关键字段` / `## 状态说明`
8. `## 权限` / `## 推荐场景`

## 写法要求

- 面向使用者，不写实现层细节。
- 只写用户需要知道的用途、操作、权限、结果和限制。
- 优先使用步骤表、字段表、状态表、权限表。

## 禁止项

- 把前端或后端实现细节写进用户指南。
- 修改正文后忘记更新时间或展示入口同步。
