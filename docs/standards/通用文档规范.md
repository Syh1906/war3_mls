# 通用文档规范

> Parent: [README.md](README.md)

## 适用范围

本规范适用于 `docs/` 下所有正式文档。临时备忘、排查草稿和本机资料放在 `.local/`，不进入正式 docs。

## 共通规则

- 先写定位，再写细节。
- 第一屏说明读者、范围和边界。
- 结构化信息优先用表格、清单或图示。
- 修改正式 docs 后，同步检查父级 `README.md`、`docs/README.md` 和相关规范或模板。
- 信息不足时标注“未验证”，不要补写未确认结论。

## 真相源

冲突时按以下顺序判断：代码和配置优先，其次是官方参考，再其次是 Markdown 文档。文档不能覆盖已确认的实现行为。

## 风格

- 使用中文。
- 语气直接，少形容词。
- 术语保持一致：MLS、云脚本、模拟器、房间、玩家、Bridge。
- 避免“简单、显然、强大、优雅、完美、深入探讨、赋能、全面”。

## docs impact review

代码、配置、脚本、接口、UI、行为、目录结构、测试口径或计划状态变化后，至少检查当前文档、父级 README、总导航、API 文档、用户指南、计划入口和 LLM 约束是否需要同步。
