# 客户端对接

客户端（War3 地图脚本、Python 测试脚本等）通过 HTTP 与模拟器通信，替代平台云脚本通道。

## 通信流程

```
客户端                          模拟器（云脚本）
  │                                │
  │  POST /api/bridge/login        │  客户端登录房间
  │──────────────────────────────→ │
  │                                │
  │  POST /api/bridge/event        │  客户端发事件给云脚本
  │──────────────────────────────→ │
  │                                │  Lua 脚本处理事件
  │                                │  调用 MsSendMlEvent 发回数据
  │                                │
  │  GET /api/bridge/poll/...      │  客户端轮询云脚本返回的事件
  │──────────────────────────────→ │
  │  ←── events JSON              │
```

## Bridge 配置

可通过 `POST /api/bridge/config` 生成配置，格式：

```lua
return {
    base_url = "http://127.0.0.1:5000",
    room_id = "room-001",
    player_index = 0,
    poll_interval = 0.05,
    req_sign_enable = false,
}
```

## Python 客户端示例

```python
import requests
import time

BASE = "http://127.0.0.1:5000"
ROOM = "room-001"
PLAYER = 0

# 登录房间
requests.post(f"{BASE}/api/bridge/login", json={
    "room_id": ROOM,
    "player_index": PLAYER,
    "name": "测试玩家"
})

# 发送事件给云脚本
requests.post(f"{BASE}/api/bridge/event", json={
    "room_id": ROOM,
    "player_index": PLAYER,
    "ename": "buy_tower",
    "evalue": '{"id": 1}'
})

# 轮询云脚本返回的事件
while True:
    r = requests.get(f"{BASE}/api/bridge/poll/{ROOM}/{PLAYER}")
    events = r.json().get("events", [])
    for ev in events:
        print(f"收到事件: [{ev['ename']}] {ev['evalue']}")
    time.sleep(0.05)
```

## curl 测试

```powershell
# 发送事件
curl -X POST http://127.0.0.1:5000/api/bridge/event -H "Content-Type: application/json" -d "{\"room_id\":\"room-001\",\"player_index\":0,\"ename\":\"testapi\",\"evalue\":\"\"}"

# 轮询事件
curl http://127.0.0.1:5000/api/bridge/poll/room-001/0
```

## 多客户端测试

多个客户端使用不同的 `player_index` 连接同一个房间即可：

- 客户端 A：`player_index = 0`
- 客户端 B：`player_index = 1`
- 客户端 C：`player_index = 2`

每个客户端独立轮询自己的事件队列，互不影响。

## 注意

Bridge 是本地测试适配层，不应直接进入正式平台发布包。
